БФТ-2. Внутреннее приложение менеджера кладбища на Android-планшете
1. Внутреннее мобильное приложение менеджера кладбища 
Описание
Мобильное приложение предназначено для использования сотрудниками кладбища (менеджерами) на территории кладбища с использованием служебных Android-планшетов.
Приложение используется для оперативной работы с картой кладбища, выбора конкретного места захоронения, внесения информации о захоронении и фиксации точных координат места с использованием спутникового модуля планшета (GNSS).
Приложение предназначено исключительно для внутреннего использования и не является публичным пользовательским приложением.
Требования
Приложение устанавливается на служебные планшеты с операционной системой Android.
Приложение предназначено для использования сотрудниками кладбища (менеджерами).
Приложение должно поддерживать работу в полевых условиях.
Основные функции приложения должны работать без подключения к интернету.
Приложение использует встроенный спутниковый модуль планшета (GNSS) для определения геопозиции.
Приложение должно корректно работать при слабом или отсутствующем сигнале мобильной сети.
2. Оффлайн-карта кладбища
Описание
Функционал предназначен для отображения карты кладбища непосредственно в приложении планшета без необходимости подключения к интернету.
Карта используется менеджером кладбища для навигации по территории кладбища и выбора конкретных участков и мест захоронений.
Требования
Карта кладбища должна храниться внутри приложения (локально).
Карта должна работать без интернет-соединения.
Должна поддерживаться возможность:
перемещения по карте;
масштабирования;
отображения участков и мест захоронений.
На карте должны отображаться статусы мест:
свободно;
забронировано;
захоронено;
резерв.
3. Определение текущего местоположения менеджера
Описание
Функционал предназначен для определения текущего местоположения менеджера на территории кладбища с использованием спутникового модуля планшета.
Определение координат осуществляется через встроенный GNSS-модуль устройства (GPS/GLONASS или аналогичные спутниковые системы).
Требования
Приложение должно получать координаты через стандартные механизмы геолокации Android.
Для определения координат используется спутниковый модуль устройства (GNSS).
Определение координат должно работать без использования сотовой связи.
На карте должна отображаться текущая позиция пользователя.
Должна быть возможность центрирования карты по текущему местоположению.
4. Выбор места захоронения на карте
Описание
Функционал позволяет менеджеру кладбища выбрать конкретный участок или место захоронения непосредственно на карте кладбища.
Требования
Менеджер может выбрать место:
нажатием на объект на карте;
по номеру участка или места.
После выбора открывается карточка места захоронения.
Карточка места доступна для просмотра и редактирования данных.
5. Фиксация координат места захоронения
Описание
Функционал предназначен для записи точных координат места захоронения непосредственно на месте проведения работ.
Координаты фиксируются с использованием спутникового модуля планшета.
Требования
При работе с карточкой места менеджер может зафиксировать координаты точки.
Координаты сохраняются в виде:
широты (latitude);
долготы (longitude).
Дополнительно сохраняются:
точность определения координат;
дата и время фиксации.
Координаты автоматически привязываются к выбранному месту захоронения.
6. Внесение данных о захоронении
Описание
Функционал позволяет менеджеру кладбища внести или обновить информацию о захоронении непосредственно на месте.
Требования
Менеджер может вносить данные в карточку места захоронения.
Данные сохраняются локально на устройстве.
Сохранение данных возможно без подключения к интернету.

7. Локальное сохранение данных и последующая синхронизация
Описание
Функционал обеспечивает сохранение всех действий пользователя на планшете при отсутствии интернет-соединения и передачу данных в центральную систему при появлении связи.
Требования
Все данные сохраняются локально на планшете.
Реализуется механизм очереди изменений.
После появления интернет-соединения данные автоматически передаются в центральную систему.
Для каждой записи фиксируется статус:
сохранено локально;
ожидает синхронизации;
синхронизировано;
ошибка синхронизации.
8. Корректировка роли «Менеджер кладбища» для работы с мобильным приложением
Описание
В связи с внедрением внутреннего мобильного приложения для Android-планшетов необходимо расширить и скорректировать роль «Менеджер кладбища» в информационной системе.
Роль должна предусматривать необходимые права доступа для работы с функционалом мобильного приложения, включая работу с картой кладбища, внесение данных о захоронениях и фиксацию координат мест захоронения.
Требования
Роль должна предоставлять доступ к следующим операциям:
просмотр карты кладбища;
просмотр информации о местах захоронений;
выбор места захоронения на карте;
внесение и редактирование данных о захоронении;
фиксация координат места захоронения;
сохранение данных с мобильного устройства.
Доступ к указанному функционалу должен быть доступен для Android приложения планшета.
Все действия, выполненные пользователем с ролью «Менеджер кладбища», должны фиксироваться в системе аудита c пометкой «действие выполнено с планшета».
